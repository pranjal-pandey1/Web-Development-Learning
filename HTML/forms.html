Explanation of attribute

  • action: Specifies the URL where the form data will be sent for processing.
  • method: Defines the HTTP method to be used when submitting the form (GET or POST).
  • label: Provides a label for the input fields, improving accessibility.
  • input type="text": Creates a single-line text input field.
  • input type="email": Creates an input field for email addresses, with built-in validation.
  • input type="password": Creates a password input field that masks the entered characters.
  • required: Indicates that the input field must be filled out before submitting the form.
  • input type="submit": Creates a submit button to send the form data.

id and name attributes are used to identify the form elements uniquely and to reference them when processing the form data. name attribute is used as the key when the form data is submitted.and id attribute is used to link the label to the input field for better accessibility.


Types of Forms Elements

  • Text Input
  • Password Input
  • Radio Buttons
  • Checkboxes
  • Dropdown Menus
  • Text Areas
  • File Uploads
  • Submit Buttons

Radio Buttons: Radio buttons allow users to select one option from a predefined set of options. They are typically used when there are multiple choices, but only one can be selected at a time.

Checkboxes: Checkboxes allow users to select one or more options from a list. They are useful when multiple selections are allowed.

Dropdown Menus: Dropdown menus provide a list of options in a compact format, allowing users to select one option from the list.



Text Areas: Text areas are used for multi-line text input, allowing users to enter longer pieces of text, such as comments or messages.




File Uploads: File upload fields allow users to select and upload files from their device.



Submit Buttons: Submit buttons are used to send the form data to the server for processing. They are typically placed at the end of the form.
