Semantic And Non Semantic Tags

Semantic HTML tags are tags that clearly describe their meaning and purpose to both the browser and the developer. They make your HTML code more readable, structured, and SEO-friendly because search engines and screen readers understand the content better.

Examples of Semantic Tags:

  • <header>: Represents the header section of a document or a section.
  • <nav>: Defines a set of navigation links.
  • <article>: Represents a self-contained piece of content that could be distributed independently.
  • <section>: Defines a section in a document, such as chapters, headers, footers, or any other sections of the document.
  • <footer>: Represents the footer section of a document or a section.
  • <aside>: Represents content that is tangentially related to the content around it, like sidebars.

Non-semantic HTML tags, on the other hand, do not provide any information about the content they contain. They are primarily used for styling and layout purposes, without conveying any meaning about the content itself.

Examples of Non-Semantic Tags:

  • <div>: A generic container for flow content that has no special meaning.
  • <span>: An inline container for phrasing content that has no special meaning.
